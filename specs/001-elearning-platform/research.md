**Feature**: `001-elearning-platform` | **Phase**: 0 — Outline & Research | **Date**: 2026-07-16

## 1. AI API Selection

### Decision: Speech-to-Text
- **Chosen**: OpenAI Whisper API (fallback: browser Web Speech API)
- **Rationale**: Whisper supports both English and Mandarin Chinese with high accuracy; available as a managed API with no self-hosting overhead for a small-scale deployment.
- **Alternatives considered**:
  - Google Cloud Speech-to-Text: More expensive for free-tier usage; slightly better Mandarin support but adds another vendor dependency.
  - Web Speech API (browser-native): Free and has zero latency overhead; limited to Chrome/Edge, no server-side transcript storage — use as fallback only.

### Decision: Chinese Tone Scoring
- **Chosen**: Post-processing pipeline on Whisper transcript using a tone detection model (e.g., Azure Cognitive Services Pronunciation Assessment, or a custom model via HuggingFace)
- **Rationale**: Whisper transcribes Chinese text but does NOT score tonal accuracy. A separate pronunciation assessment API is required per constitution principle VI.
- **Open item**: The specific provider (Azure vs. HuggingFace) is deferred to Sprint 3. The `speaking_analysis.suggestions` JSONB field is flexible enough to hold tone feedback from any provider.
- **Alternatives considered**:
  - iFlytek/Xunfei API: Best-in-class for Mandarin pronunciation; requires Chinese corporate account.
  - Azure Cognitive Services Pronunciation Assessment: Supports Mandarin tones natively; available on Azure Free Tier (5 hours/month).

### Decision: Grammar & Writing Analysis
- **Chosen**: Google Gemini API (primary); OpenAI GPT-4o-mini (fallback)
- **Rationale**: Gemini API has a generous free tier suitable for < 50 users; both support structured JSON output for `grammar_errors` and `suggestions` fields.
- **Alternatives considered**: Claude API — excellent quality but no free tier for production use.

## 2. AWS S3 File Storage Strategy

### Decision: S3 Lifecycle Rule for Audio Deletion
- **Chosen**: S3 Object Lifecycle Rule on `recordings/` prefix with `Expiration: Days=15`
- **Rationale**: Automated deletion requires no application-level cron; S3 handles the physical deletion. A backend cron job (or AWS Lambda triggered by S3 event notification) updates `speaking_recordings.is_deleted = true` after the file is removed.
- **Implementation details**:
  - Lifecycle rule configured via Terraform or AWS console on the recordings/ prefix
  - S3 Event Notification (ObjectRemoved) → Lambda → PATCH `/internal/recordings/{id}/mark-deleted`
  - Alternatively: scheduled cron job queries `WHERE expires_at < now() AND is_deleted = false` and checks S3 existence before updating the flag.

### Decision: Pre-signed URLs for Upload
- **Chosen**: Pre-signed PUT URLs generated by the backend before the client uploads
- **Rationale**: Client-side direct upload to S3 avoids proxying large files through the backend; the backend validates file metadata (size, type) BEFORE generating the pre-signed URL, satisfying the dual-validation requirement.
- **Flow**: Client → Backend (validate size/type) → S3 pre-signed URL → Client uploads directly → Backend confirms via S3 HeadObject.

## 3. PostgreSQL Database Configuration

### Decision: Database Encoding & Collation
- **Chosen**: `UTF8` encoding; collation `en_US.utf8` (fallback if `zh_CN.utf8` locale unavailable on host)
- **Rationale**: UTF8 stores all Unicode characters including CJK correctly. `zh_CN.utf8` improves ORDER BY for Chinese strings but is optional; `en_US.utf8` is universally available.
- **Create command**:
  ```sql
  CREATE DATABASE elearning_db
    WITH ENCODING 'UTF8'
    LC_COLLATE='zh_CN.utf8'
    LC_CTYPE='zh_CN.utf8'
    TEMPLATE=template0;
  ```
  If locale unavailable: replace with `'en_US.utf8'`.

### Decision: Full-Text Search for Chinese
- **Deferred to post-MVP**: `pg_trgm` extension for trigram search on `flashcards.term` (supports CJK with `zhparser` extension).
- **Rationale**: Not required for v1; standard ILIKE queries are sufficient for < 10,000 flashcards.

## 4. Frontend Chinese Support

### Decision: CJK Font Loading
- **Chosen**: Google Fonts `Noto Sans SC` (Simplified Chinese) loaded via `<link>` in `index.html`
- **Rationale**: Noto Sans SC covers all CJK Unified Ideographs; free; CDN-hosted for fast delivery.
- **Fallback stack**: `"Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`

### Decision: Pinyin Input & Display
- **Chosen**: Standard HTML `<input type="text">` with IME support (OS-level); no custom input method library.
- **Rationale**: Modern browsers handle IME input correctly for Chinese; the `compositionstart`/`compositionend` events allow React to defer state updates until IME composition is complete, avoiding encoding glitches on submit.

### Decision: HSK Level Taxonomy
- **Chosen**: `exercises.level` stores HSK 1–6 for Chinese content, CEFR A1–C2 for English; the UI renders a different selector based on `language`.
- **Rationale**: Field is `VARCHAR(20)` in the schema — flexible enough for both taxonomies without schema changes.

## 5. Authentication & Security

### Decision: JWT Strategy
- **Chosen**: Short-lived access token (15 min) + long-lived refresh token (7 days) stored in httpOnly cookies
- **Rationale**: httpOnly cookies prevent XSS theft of tokens; refresh token rotation on each use prevents replay attacks.
- **Alternatives considered**: LocalStorage for tokens — rejected due to XSS vulnerability.

### Decision: OTP Reset Flow
- **Chosen**: 6-digit OTP, 10-minute TTL, stored in `password_reset_tokens`; single-use (`is_used = true` after first verification)
- **Rationale**: Matches the `password_reset_tokens` schema; prevents OTP replay attacks.
